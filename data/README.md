# Данные

## Откуда брать данные

Данные для каждого трека публикуются организаторами хакатона:

- [Задание трек 1](../../tasks/task-01.md) — mrxs-сканы
- [Задание трек 2](../../tasks/task-02.md) — раковые регистры
- [Задание трек 3](../../tasks/task-03.md) — корпус для ассистента

Следите за обновлениями в [репозитории организаторов](../../README.md).

## Структура папки

```
data/
├── raw/           # Исходные данные (не изменять)
├── processed/     # Подготовленные данные
└── README.md
```

## Важно

- **Не коммитьте** в git большие файлы (mrxs, полные регистры, эмбеддинги)
- Используйте `.gitignore` или Git LFS, если организаторы это разрешают
- В README репозитория укажите, как скачать данные для воспроизведения
- Соблюдайте условия использования и [ETHICS.md](../ETHICS.md)

## По трекам

### Трек 1: mrxs-сканы

- Формат: Mirax / mrxs (+ сопутствующие файлы папки)
- Типичный размер: гигабайты — хранить локально или в облаке
- Инструменты: OpenSlide, специализированные SDK вендора

### Трек 2: Раковые регистры

- Формат: обычно CSV / Parquet
- Проверьте псевдонимизацию: не публикуйте идентифицирующие поля
- Опишите схему в [docs/data_schema.md](../docs/data_schema.md)

### Трек 3: Научный корпус

- PDF, JSON, PubMed-выгрузки
- Для RAG: чанки и индекс — в `data/processed/` или внешнее хранилище
- API-ключи — только в `.env`, не в репозитории
