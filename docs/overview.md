# Обзор решения

## Выбранное задание

**Трек 3** — AI-ассистент для научных исследований ([task-03](../tasks/task-03.md)).

## Проблема

Врач-онколог при принятии клинических решений сталкивается с проблемой информационной перегрузки:

- В PubMed индексируется ~30 миллионов публикаций, ежегодно добавляется ~1.5 млн новых
- По узким клиническим вопросам (например, "осимертиниб при T790M после прогрессии") выходит десятки новых статей в год
- Качество публикаций сильно варьируется: от РКИ фазы 3 в NEJM до preclinical-исследований и нарративных обзоров
- Разные исследования измеряют разные исходы, используют разные критерии включения и методики

Систематический обзор по правилам Cochrane занимает у профессионального исследователя **6-12 месяцев**. На рутинный клинический вопрос врач не может потратить такое время, но и принимать решение на основе одной случайно найденной статьи — недопустимо.

## Цель

Создать AI-ассистента, который за 1-2 минуты:

1. Структурирует свободный вопрос врача в формат PICO
2. Проводит поиск в PubMed (не имитирует — реальный API)
3. Оценивает качество найденных источников по принятым в EBM методикам
4. Картирует научное поле (Scoping Review)
5. Оценивает возможность метаанализа
6. Формирует доказательный отчёт со ссылками на конкретные PMID, без выдумывания данных

Результат должен быть **повторяем, прозрачен и поддаётся аудиту** — каждое утверждение в отчёте можно проверить по конкретной статье.

## Сценарии использования

### Сценарий 1: подготовка к клиническому решению

Врач-онколог получает пациента с распространённым NSCLC, мутация EGFR T790M после прогрессии на первой линии терапии (гефитиниб). Перед выбором терапии второй линии — спрашивает ассистента:

> *Какова эффективность osimertinib при NSCLC с мутацией EGFR T790M после прогрессии на гефитинибе?*

Получает структурированный отчёт с HR 0.30 для ВБП, ORR 71% vs 31%, обсуждением crossover в анализе общей выживаемости. Ссылки на AURA3 (NEJM, 2017), финальный анализ ОВ (J Clin Oncol, 2020), японскую подгруппу.

### Сценарий 2: подготовка к научному обсуждению

Молодой исследователь готовится к презентации на онкологическом конгрессе по теме адъювантной иммунотерапии при гепатоцеллюлярной карциноме. Просит:

> *Эффективность atezolizumab плюс bevacizumab при гепатоцеллюлярной карциноме*

Получает обзор IMbrave150 и IMbrave050 с конкретными данными по ОВ, ВБП, RFS, обсуждением гомогенности популяций и возможности метаанализа.

### Сценарий 3: поиск пробелов в исследованиях

Аспирант выбирает тему диссертации. Спрашивает ассистента, какие пробелы есть в текущих исследованиях по конкретной нозологии. Scoping Review агент возвращает структурированный список — какие популяции изучены недостаточно, какие комбинации не сравнивались, какие исходы редко измеряются.

## Подход

**Архитектура:** мультиагентный LLM-пайплайн (6 специализированных агентов) поверх Yandex AI Studio с использованием реального PubMed E-utilities API.

**Ключевой принцип:** все факты в отчёте — из абстрактов реальных PubMed-статей. LLM не выдумывает PMID, цифры, авторов. Если данных нет — система прямо говорит "недостаточно данных".

**Контроль галлюцинаций:**
- Verifier генерирует поисковые запросы, не выдаёт ответы
- Researcher возвращает только то, что есть в PubMed
- Critic оценивает по чек-листам (RoB 2.0, ROBINS-I и т.д.) на основе абстрактов
- Synthesizer обязан опираться только на переданные ему источники с PMID

Подробнее — [architecture.md](architecture.md).

## Социальный эффект

- **Доступность доказательной медицины:** не все клиники имеют доступ к подписке Cochrane Library, UpToDate ($500+/год). PubMed бесплатен и открыт, наш ассистент — тоже.
- **Снижение информационной перегрузки врача:** время на анализ литературы сокращается с дней до минут.
- **Поддержка для региональных врачей:** там, где нет онкологического консилиума с биостатистом, ассистент даёт структурированный второй взгляд.
- **Образовательная ценность:** ординаторы и аспиранты видят, как должен быть устроен систематический обзор, какие критерии оценки качества применяются.

## Ограничения и риски

### Что ассистент НЕ делает

- **Не заменяет врача.** Не выдаёт клинических рекомендаций. Все отчёты содержат явный дисклеймер.
- **Не диагностирует и не назначает лечение.** Только систематизирует литературу.
- **Не рассчитывает метаанализ.** Только оценивает его возможность (это требует биостатистической экспертизы).
- **Не учитывает индивидуальные особенности пациента** — сопутствующие заболевания, противопоказания, доступность препаратов.

### Где может ошибаться

- **Поиск только по PubMed.** Embase, Cochrane Library, региональные базы не включены — могут быть упущены публикации.
- **Только английский поиск.** Русскоязычные публикации не попадают в выдачу.
- **Только абстракты.** Полные тексты не анализируются (требует доступа к платным журналам).
- **Зависимость от качества LLM.** Yandex AI Studio может неправильно классифицировать тип исследования при нестандартных абстрактах.
- **Предвзятость публикаций.** Negative trials публикуются реже, что искажает реальную картину доказательной базы.

### Почему решение нельзя напрямую применять в клинике

1. Не сертифицировано как медицинское изделие (Roszdravnadzor, FDA, CE Mark)
2. Не валидировано на независимом наборе клинических вопросов
3. Не интегрировано с электронной медицинской картой
4. Этическая ответственность за решение — на враче, а не на алгоритме
