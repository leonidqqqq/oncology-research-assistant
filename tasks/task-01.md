# Задание 1: Виртуальная гистотопограмма

**Трек:** автоматическая реконструкция цифровых макропрепаратов из нескольких mrxs-сканов.

## 1. Контекст и практическая проблема

В патоморфологической практике материал хирургической резекции в подавляющем большинстве случаев превышает размер стандартной гистологической кассеты (35×25 мм). Один объект резекции разделяют на несколько (до 30–60) частей в нескольких (10–15) пластинах. После гистологической проводки и микротомии врач-патологоанатом вынужден реконструировать гистологические препараты, собирая их как пазл. В цифровом виде после сканирования врач получает набор разрозненных `.mrxs`-файлов — частей одной пластины.

### Проблемы сегодня

1. Врач мысленно собирает пазл из нескольких цифровых слайдов; ручное сопоставление утомляет.
2. Разрывы между файлами мешают оценить непрерывность патологического процесса и протяжённость опухоли для стадирования.
3. Сборка из 6, 8 или 12+ кассет вручную — крайне сложная задача с риском потери качества и геометрии стыков.

Большинство просмотрщиков цифровых гистологических препаратов не предоставляют встроенных инструментов для пространственной реконструкции и бесшовной сшивки. Готового доступного ПО для автоматической сборки массивов сканов в формате `.mrxs` на текущий момент нет.

## 2. Цель кейса

Разработать прототип, который позволяет:

- **Импортировать данные:** загружать несколько (от 2 до 12) файлов формата `.mrxs`.
- **Определять взаимное расположение:** в автоматическом, ручном и полуавтоматическом режиме (контрольные точки).
- **Выполнять сшивку:** объединять сканы в единое изображение с минимизацией искажений и артефактов на стыках. Сшивка — в едином масштабе, **без** автоматического «дорисовывания» нестыкующихся областей.
- **Формировать единый объект:** виртуальную цифровую гистотопограмму высокого разрешения.
- **Экспортировать результат:** в открытый стандартизированный формат, пригодный для просмотра, аннотирования и анализа.

## 3. Требования к определению взаимного расположения

Без реализации этой базовой функции решение **не будет принято**.

### 3.1. Ручная разметка и корректировка стыков (обязательный режим)

- Ручная регистрация: пары контрольных точек на смежных участках препаратов.
- Интерактивная коррекция: сдвиг, поворот, масштабирование отдельных сканов при ошибках алгоритма.

### 3.2. Полуавтоматическое определение соседства по цветовой маркировке

На краях фрагментов в одной пластине маркируются гистологической тушью разных цветов. Программа должна:

- находить цветные метки на краях сканов;
- сопоставлять метки между кассетами;
- предлагать схему сборки;
- давать врачу подтвердить или поправить схему перед сшивкой.

### 3.3. Автоматическая точная регистрация и бесшовная сшивка

После предварительного определения соседства и согласования с оператором — автоматическое уточнение взаимного расположения.

### 3.4. Сохранение результата

- Итоговый макропрепарат со всеми уровнями увеличения (от общего вида до отдельных клеток).
- Сохранение в открытый формат, открывающийся в стандартных просмотрщиках (Pannoramic Viewer, QuPath и др.).

## 4. Дополнительные требования

- **Скорость:** обработка 3–5 сканов — не больше 10–15 минут на обычном рабочем компьютере.
- **Интерфейс:** понятный, с индикатором прогресса.
- **Устойчивость:** при ошибках (нет перекрытия, повреждённый файл) — понятное сообщение, без «падения».
- **Контейнер** с программой должен без модификации кода запускаться на любой хостовой ОС.

## 5. Что предоставляют организаторы

Реальные данные из практики НМИЦ онкологии им. Н.Н. Петрова.

| Параметр | Значение |
|----------|----------|
| Формат | `.mrxs` (3DHISTECH) |
| Размер файла | 0,5–2 ГБ |
| Кейсов | 5–10 наборов по 3–12 кассет |
| Стыковка | «встык»: опора на цветовые маркеры или ручной ввод точек |
| Маркировка | чёрный, синий, зелёный, красный (на части сканов) |

**Два набора данных:**

- **Публичный** (2–3 кейса) — для отладки.
- **Скрытый** (3–5 кейсов) — для финальной оценки жюри.

Для каждого кейса: схема расположения кассет, разметка контрольных точек, заключение врача о допустимости качества стыковки.

## 6. Что сдаёт команда

1. **Рабочая программа** — готова к запуску без дополнительных настроек.
2. **Исходный код** с инструкцией по запуску.
3. **Документация** — как работает, какие алгоритмы, установка и применение.
4. **Примеры работы** — видео 3–5 минут; 2–3 готовых макропрепарата в открытом формате.
5. **Презентация / видео-питч** — 5–7 минут.

## 7. Ожидаемый итог

Рабочий воспроизводимый прототип: определение расположения фрагментов → точная сшивка → экспорт в стандартный файл для диагностической практики.
