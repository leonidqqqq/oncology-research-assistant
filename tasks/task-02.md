# Задание 2: Прогноз заболеваемости и выживаемости

**Трек:** анализ заболеваемости с использованием синтетических данных и анонимизации субъектов.

Разработка приложения для построения ML-моделей прогноза заболеваемости, смертности и выживаемости на основе данных популяционных раковых регистров. Участники определяют, какие признаки необходимы для надёжной модели.

## Введение

Популяционные раковые регистры — ключевой источник для оценки онкологической ситуации в регионе: заболеваемость, смертность, выживаемость, структура диагнозов, распределение по полу, возрасту, стадии и морфологическому типу.

Данные важны для планирования лекарственной терапии, оборудования, кадров и маршрутизации пациентов. На практике регистры часто используются для ретроспективной отчётности, тогда как для управления здравоохранением критичны **прогнозы** на ближайшие годы.

## Проблематика

- Ограниченность ретроспективной аналитики.
- Сложность планирования ресурсов без прогноза.
- Недостаточная индивидуализация прогнозов (возраст, пол, стадия, морфология, локализация).
- Риск управленческих решений «вслепую».
- Необходимость **интерпретируемости** — важно понимать факторы риска, а не только численный прогноз.

## Цель кейса

Воспроизводимый аналитический прототип на данных популяционного ракового регистра, который позволяет:

1. прогнозировать заболеваемость злокачественными новообразованиями;
2. прогнозировать смертность;
3. оценивать и прогнозировать выживаемость;
4. определить набор признаков для надёжной модели и оценить их качество в датасете;
5. предложить применение результатов для планирования ресурсов онкологической службы.

## Решение

Приложение для табличных медицинских данных с временными прогнозами на уровне популяции и/или групп пациентов (ML).

**Базовое решение включает:**

- подготовку и очистку данных;
- анализ структуры регистра;
- выделение значимых признаков;
- интерактивное взаимодействие с врачом (типы признаков, цели прогноза, выбросы, тренды, сезонность);
- построение ML-моделей заболеваемости, смертности и/или выживаемости;
- оценку качества и интерпретацию;
- визуализацию выводов;
- описание применения в управлении онкологической службой.

**Формат:** интерактивный дашборд, аналитический веб-прототип с загрузкой данных и расчётом метрик, или воспроизводимый ML-проект с документацией.

**Ключевое требование:** решение и модели должны быть содержательно осмысленными с точки зрения онкоэпидемиологии и управления здравоохранением.

## Прогнозируемые показатели

Команда строит модель(и) для одного или нескольких показателей:

- заболеваемость;
- смертность;
- общая выживаемость;
- выживаемость по стадии, полу, возрасту;
- динамика показателей во времени.

Важно объяснить: какие признаки использованы, какие значимы, ограничения модели, надёжность прогноза, какие выводы можно и нельзя делать.

## Данные

Обезличенные данные популяционного ракового регистра (ссылки предоставляет организатор).

## Аналитические шаги

### 1. Анализ и подготовка данных

Полнота и качество, пропуски, дубликаты, даты, возраст на момент диагноза, горизонт наблюдения, переменные для survival analysis, ограничения датасета.

### 2. Формирование признаков

Возраст, пол, год диагноза, локализация, морфология, стадия, время до смерти/цензирования, статус события, календарный период, агрегаты по региону.

### 3. Построение моделей

ML-подходы с обоснованием выбора. Простая интерпретируемая модель может быть ценнее сложной без понятной логики.

### 4. Оценка качества

Concordance index, time-dependent AUC, Brier score, MAE/RMSE/MAPE, calibration plots, сравнение с историей, устойчивость на отложенной выборке.

### 5. Интерпретация и визуализация

Группы риска, динамика, важность признаков, сценарии нагрузки, управленческие выводы — графики, таблицы, дашборд или отчёт.

## Технологический стек

- Стек — на выбор команды.
- **Docker** для воспроизводимого запуска — обязательно.

## Что сдаёт команда

1. Рабочий прототип (Docker, полностью собранный проект).
2. Исходный код со структурой и инструкцией.
3. Документация: данные, признаки, модели, метрики, ограничения, интерпретация.
4. Примеры: прогнозы, графики/дашборд, таблица результатов, интерпретация для врача/управленца.
5. Презентация / видео-питч — 5–7 минут.

## Ожидаемый итог

Приложение для прогнозирования онкоэпидемиологических показателей и поддержки управленческих решений: загрузка → базовые показатели → признаки → модель → оценка качества → интерпретация → визуализация → практическое применение.
